Git Branch Commands

Branches in Git allow you to work on different parts of a project separately. Here are the basic commands to manage branches:

1. Create a New Branch

git branch [branch-name]

This command creates a new branch. Replace [branch-name] with the name you want for your branch.

2. Switch to a Branch

git checkout [branch-name]

Switches to the specified branch so you can work on it.

3. Create and Switch to a New Branch

git checkout -b [branch-name]

Creates a new branch and switches to it in one step.

4. List All Branches

git branch

Displays all branches in your repository. The current branch is highlighted with an asterisk (*).

5. Delete a Local Branch

git branch -d [branch-name]

Deletes a branch that has been merged. Use -D to force delete an unmerged branch.

6. Delete a Remote Branch

git push origin --delete [branch-name]

Deletes a branch from the remote repository.

Helpful Tips

  • Always name branches descriptively (e.g., feature-login, bugfix-header).
  • Keep your branches up-to-date with git pull.
  • Use git merge to combine branches when your work is done.

For more details, visit the official Git documentation.
