Introduction to Git

Git is a distributed version control system that helps developers manage their codebase. Here are some commonly used Git commands:

Basic Commands

  • Initialize a repository: 
    git init

    Start a new Git repository in the current folder.

  • Clone a repository: 
    git clone [repository URL]

    Copy a remote repository to your local machine.

  • Check the status of the repository: 
    git status

    See the current state of your files and changes.

  • Add changes to the staging area: 
    git add [file]

    Prepare a specific file for the next commit.

    git add . (Add all files)

    Stage all changes in the repository.

  • Commit changes: 
    git commit -m "Your commit message"

    Save your changes with a message describing them.

  • Push changes to a remote repository: 
    git push

    Send your committed changes to a remote server.

  • Pull changes from a remote repository: 
    git pull

    Download and merge changes from a remote repository.

  • Create a new branch: 
    git branch [branch-name]

    Make a new branch to work on separate changes.

  • Switch to another branch: 
    git checkout [branch-name]

    Move to a different branch in your repository.

  • Merge branches: 
    git merge [branch-name]

    Combine changes from one branch into another.

Helpful Tips

  • Always write meaningful commit messages.
  • Keep your branches up-to-date by pulling changes regularly.
  • Use git log to view commit history.
  • To undo the last commit (without losing changes): 
    git reset --soft HEAD~1

For more information, visit the official Git documentation.
